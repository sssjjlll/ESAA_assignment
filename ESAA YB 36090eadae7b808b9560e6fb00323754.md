### 새롭게 알게 된 내용

- `corr()` + heatmap으로 변수 간 중복 정보 확인 (다중공선성 가능성)
- 범주형인 type(종류) 변환이 모델 성능에 직접 영향을 주며, 숫자형 변수만 다루는 모델을 위해 인코딩 과정이 필수

### 어려운 점

- 각 변수 분포를 quality별로 비교할 때, 클래스(quality) 불균형이 있으면 시각적으로 왜곡되어 보일 수 있고, 어떤 지표로 “차이가 크다/작다”를 판단할지 애매할 수 있다.
- 전처리(스케일링/이상치 처리/로그 변환 등)를 어디까지 해야 “과한 가공”이 아닌지, 그리고 성능 향상에 실제로 도움이 되는지 판단하기 어렵다.

### 배울 점

- 모델링 전에 type 인코딩 방식과 모델을 세트로 고민하기:
    - 선형/거리 기반 모델: 원-핫 + 스케일링 고려
    - 트리 기반 모델: 라벨 인코딩도 가능, 다만 검증으로 확인
- 단순 1회 학습/평가가 아니라, 교차검증(K-Fold)과 성능 지표(예: RMSE/MAE)를 명확히 정해서 안정적으로 비교하는 흐름을 만들기
